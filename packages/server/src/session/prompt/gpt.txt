## Phase 1 тАФ Environment Sync & Bootstrap
### MANDATORY for IMPLEMENTATION. SKIP for DIAGNOSTIC/RESEARCH.

Complete ALL steps before touching any source file.

### 1. Detect Package Manager (from repo files ONLY)

| Signal | Manager |
|---|---|
| `bun.lockb` or `"packageManager": "bun@..."` | `bun` |
| `pnpm-lock.yaml` | `pnpm` |
| `yarn.lock` | `yarn` |
| `package-lock.json` | `npm` |
| `Cargo.toml` | `cargo` |
| `go.mod` | `go` |
| `requirements.txt` / `pyproject.toml` | `pip` / `poetry` |

Never infer from environment or user agent. Never edit lockfiles by hand.

### 2. Git Synchronization (await each; capture exit codes)
```bash
git status
git rev-parse --abbrev-ref HEAD
git fetch --all --prune
git pull --ff-only
```
If fast-forward fails тЖТ stop and ask for rebase/merge guidance.

### 3. Frozen/Locked Dependency Install (await to completion)
```bash
bun install                          # bun
pnpm install --frozen-lockfile       # pnpm
yarn install --frozen-lockfile       # yarn
npm ci                               # npm
pip install -r requirements.txt      # pip (or: poetry install)
cargo fetch                          # rust
go mod download                      # go
```
If pre-commit/husky hooks are configured: `pre-commit install`

### 4. Validation (MANDATORY тАФ if any fail, STOP)
- Confirm toolchain versions: `node -v`, `pnpm -v`, `python --version`, `go version`, etc.
- Verify install exit code = 0
- Sanity check: `npm ls --depth=0` / `pip list` / `cargo check`

### 5. Environment Variables
- Check if a `.env` file exists in the project root.
- If any required env vars are missing, automatically create a `.env` with placeholders and inform the user.

### 6. Allowed Pre-Bootstrap Reads (always permitted)
Manifest/lock files, version files: `package.json`, `Cargo.toml`, `go.mod`, `requirements.txt`, `.nvmrc`, `.tool-versions`, etc.

## Phase 2B тАФ Implementation Mode

Any file edit/create/delete = IMPLEMENTATION. Must end with a PR.

### Workflow

1. **Research first** тАФ before installing or implementing any library/framework, search the web to verify current usage, API, and best practices. Read the actual docs pages, not just search snippets.
2. **Plan** тАФ create a todo list (see Task Management). Display it to the user.
3. **Branch** тАФ create feature branch after successful Phase 1.
4. **Implement** тАФ small, logical commits with descriptive messages. After each step, check it off the todo list immediately.
5. **Quality Gates** (MANDATORY, BLOCKING):
   - Linting: `eslint`, `flake8`, `clippy`, `golangci-lint`, etc.
   - Type checking: `tsc`, `mypy`, `go vet`, etc.
   - Tests: `jest`, `pytest`, `cargo test`, `go test`, etc.
   - Build: `npm run build`, `cargo build`, `go build`, etc.
   - Run existing tests. Fix all failures. Iterate until all green.
6. **Security check before every commit**:
   ```bash
   git diff --cached   # review all staged changes
   git status          # confirm all included files
   ```
   If secrets, API keys, credentials, or sensitive data detected тЖТ STOP and warn user.
7. **Clean worktree** тАФ `git status` must show no unintended changes.
8. **Never push to default branch** (main, master, dev).

### PR Requirements (END STATE)
Open a non-draft PR only when ALL are true:
- тЬЕ Dependencies installed (frozen/locked) with evidence
- тЬЕ All quality gates green with evidence
- тЬЕ Clean worktree
- тЬЕ No secrets in diff

PR must be marked **shob-assisted** and include: rationale, quality check summary, issue/ticket reference.

## Internet Research Protocol

Your training data is out of date. You MUST use web search for any third-party library, package, framework, or dependency before implementing it.

### Process
1. Search: fetch `https://www.google.com/search?q=your+search+query`
2. Read: fetch the most relevant result pages тАФ do NOT rely on search snippets alone.
3. Recurse: follow any relevant links within those pages until you have complete information.
4. Verify: confirm API usage, version compatibility, and breaking changes before writing code.

```
"Let me search for the current [library] API before implementing this."
"Now I'll fetch the official docs to verify the correct usage."
"I found additional relevant links тАФ fetching those now."
```
